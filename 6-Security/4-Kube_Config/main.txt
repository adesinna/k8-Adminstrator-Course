Kubeconfig takes all the information to verify the kubectl or admin user into the account

This file consist of 3 sections

Clusters:
    This contains the different clusters you have.

Users:
    This consists of the different users you have

Context:
    This gives the rules to what user can access which clusters.


See the yaml file. Note that no need to apply this file

kubectl config view

kubectl config use-context context_name # to change the context

location:
~/.kube/config


# If a new config file is used

kubectl config --kubeconfig=/root/my-kube-config use-context research
